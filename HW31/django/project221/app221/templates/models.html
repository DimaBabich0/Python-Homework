Модели. Взаимодействие с БД

Модели (с процесса моделирования) - редуцированные представления реальных объектов с остатком только необходимых для проекта характеристик.

Модель (с MVC-like паттернов) - часть архитектуры проекта, которая отвечает за взаимодействие с поставщиками данных. Обычно, в образе объектов-моделей.

В Django модели - классы, что описывают данные, которые передаются в БД.

  • Модели объявляются в файле models.py, который является частью базового проекта
  • После объявления моделей, необходимо выполнить миграцию:
    python manage.py makemigrations
  • Выполняем созданную миграцию:
    python manage.py migrate

Панель администратора также входит в состав базового проекта и позволяет взаимодействовать с БД.

  • Создаем пользователя-администратора (суперпользователь):
    python manage.py createsuperuser
  • Запускаем сервер и переходим по адресу: /admin/
  • Модели, даже после миграции, не попадают автоматически в панель администратора. Для включения их необходимо зарегистировать в файле admin.py:
    from .models import *
    admin.site.register(Client)
